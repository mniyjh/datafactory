## 〇、环境模型（必读）

系统有两种截然不同的环境策略：

| 模块 | 模型 | 运行时解析 |
|------|------|-----------|
| **数据库管理** | 各环境独立 | 每个环境(DEV/TEST/PROD)各有独立的连接配置，设 `is_current=1` 即生效。执行时使用**任务执行环境**的版本 |
| **三方API管理** | 各环境独立 | 同上。执行时使用**任务执行环境**的版本 |
| **脚本管理** | DEV→TEST→PROD 晋升链 | 执行时**始终拉取 PROD 环境已发布(`publish_status=1`)的当前版本**，与任务执行环境无关 |
| **任务管理** | DEV→TEST→PROD 晋升链 | 执行时任务内引用的脚本/子任务始终用 PROD 版本 |

> **关键结论**：脚本在 DEV 环境编辑完成后，必须逐级晋升到 PROD 并发布，才能在任务中正常执行。

**新建脚本**：

| UI 字段 | 值 |
|---------|-----|
| 脚本编码 | `PY_WEATHER_ANALYZE` |
| 脚本名称 | `天气出游分析脚本` |
| 脚本类型 | `PYTHON` |
| 状态 | `启用` |
| 描述 | `解析天气JSON综合评分判断是否适合出游` |

**DEV 版本编辑**：

| UI 字段 | 值 | 说明 |
|---------|-----|------|
| 变更说明 | `初始化` | **必填** |
| 工作目录 | `.` | 当前目录。不要填 `/tmp`（Windows 下会解析为 `C:\tmp` 报错） |
| 解释器路径 | (留空) | 留空默认 `python`（从 PATH 查找）。不要填 `/usr/bin/python3`（Windows 无此路径） |
| 脚本代码 | 见下方 | — |
| 超时时间(秒) | `60` | — |
| 重试次数 | `0` | — |
| 依赖库 | `[]` | 仅标准库，无需额外依赖 |
| 环境变量 | `{}` | — |
| 描述 | (留空) | — |

> **为什么工作目录和解释器路径填 `.` 和留空**：Python gRPC 执行时这两个值根本不传给 gRPC Server；Shell 执行时 `ProcessBuilder.directory()` 会把 `/tmp` 解析为 `C:\tmp`（Windows 下不存在），导致 `IOException`。填 `.` 和留空在 Windows/Linux 下都安全。

**脚本代码**：

```python
import json

class Script:
    def execute(self, weather_raw="", city=""):
        """
        v3: 城市名从输入参数获取，不再从API响应中提取坐标。
        """
        data = json.loads(weather_raw) if isinstance(weather_raw, str) else weather_raw

        # API 返回格式为 {"statusCode":200, "body":"..."}，需解包
        if isinstance(data, dict) and "body" in data:
            w = json.loads(data["body"]) if isinstance(data["body"], str) else data["body"]
        else:
            w = data

        c = w.get("current_condition", [{}])[0]
        temp   = int(c.get("temp_C", 0))
        desc   = c.get("weatherDesc", [{}])[0].get("value", "未知")
        wind   = int(c.get("windspeedKmph", 0))
        hum    = int(c.get("humidity", 0))
        precip = float(c.get("precipMM", 0))
        uv     = int(c.get("uvIndex", 0))

        score = 100; warns = []
        dl = desc.lower()
        if any(x in dl for x in ["rain","drizzle","shower","雨"]):    score-=30; warns.append("有降水")
        if any(x in dl for x in ["snow","sleet","hail","雪","冰雹"]): score-=40; warns.append("有降雪/冰雹")
        if any(x in dl for x in ["fog","mist","霾","雾"]):            score-=15; warns.append("能见度低")
        if   temp < 5:  score-=25; warns.append("温度过低")
        elif temp < 12: score-=10; warns.append("温度偏凉")
        elif temp > 37: score-=30; warns.append("温度过高")
        elif temp > 32: score-=10; warns.append("温度偏高")
        if   wind > 40: score-=20; warns.append("风力过大")
        elif wind > 25: score-=10; warns.append("风力较大")
        if precip > 5: score-=20; warns.append("降水量大")
        if uv > 7:     score-=10; warns.append("紫外线强")

        score = max(0, score)
        if   score >= 80: sug = "非常适合出游，祝旅途愉快！"
        elif score >= 60: sug = "适合出游，建议做好防护措施"
        else:             sug = "不太适合出游，建议改期或选择室内活动"

        return {
            "城市": city or "未知",
            "温度_摄氏度": temp, "体感温度_摄氏度": int(c.get("FeelsLikeC",temp)),
            "天气状况": desc, "风速_kmh": wind, "湿度_百分比": hum,
            "降水量_mm": precip, "紫外线指数": uv, "综合评分": score,
            "是否适合出游": "是" if score>=60 else "否",
            "出游建议": sug,
            "风险提示": "；".join(warns) if warns else "无明显风险"
        }
```

> **gRPC 执行约束**：①必须定义 `class Script` ②必须定义 `execute` 方法 ③参数名=节点 paramCode ④返回 dict。不需要 `main` / `stdin` / `print`。

**脚本代码**：

```python
import json

class Script:
    def execute(self, score="", city=""):
        """
        v3: 将天气分析结果格式化为人类可读的出行通知。
        city 直接从上游获取，不再从分析结果中提取（避免显示坐标）。
        使用简洁文本格式，兼容 Web UI 显示。
        """
        # 解析分析结果
        data = json.loads(score) if isinstance(score, str) else score

        # 引擎会包装一层，逐层解包找到真正的分析数据
        if isinstance(data, dict) and "result" in data:
            data = data.get("result", data)
        if isinstance(data, dict) and "stdout" in data:
            try:
                inner = json.loads(data["stdout"])
                if isinstance(inner, dict):
                    data = inner
            except (json.JSONDecodeError, TypeError):
                pass

        temp    = data.get("温度_摄氏度", "--")
        feels   = data.get("体感温度_摄氏度", "--")
        desc    = data.get("天气状况", "未知")
        wind    = data.get("风速_kmh", "--")
        hum     = data.get("湿度_百分比", "--")
        precip  = data.get("降水量_mm", 0)
        uv      = data.get("紫外线指数", "--")
        score_n = data.get("综合评分", 0)
        suit    = data.get("是否适合出游", "未知")
        advice  = data.get("出游建议", "")
        risks   = data.get("风险提示", "无")

        # 天气图标
        dl = desc.lower()
        if any(w in dl for w in ["sunny","clear","晴"]):   icon = "☀️"
        elif any(w in dl for w in ["cloud","多云","阴"]):  icon = "⛅"
        elif any(w in dl for w in ["rain","drizzle","shower","雨"]): icon = "🌧️"
        elif any(w in dl for w in ["snow","雪"]):           icon = "❄️"
        elif any(w in dl for w in ["fog","mist","haze","雾","霾"]): icon = "🌫️"
        else:                                               icon = "🌤️"

        # 评分等级
        if score_n >= 80:   stars, level = "⭐⭐⭐⭐⭐", "极佳"
        elif score_n >= 60: stars, level = "⭐⭐⭐⭐",   "良好"
        elif score_n >= 40: stars, level = "⭐⭐⭐",     "一般"
        else:               stars, level = "⭐⭐",       "较差"

        # 紫外线
        uv_labels = {0:"无",1:"极低",2:"极低",3:"低",4:"低",5:"中等",6:"中等",7:"高",8:"高",9:"很高",10:"很高",11:"极高"}
        uv_level = uv_labels.get(int(uv) if isinstance(uv, (int, float)) else 6, "中等")

        lines = []
        lines.append("")
        lines.append("========== 🌍 出行通知报告 ==========")
        lines.append("")
        lines.append(f"  📍 目的地: {city or '未知'}")
        lines.append("")
        lines.append("  ------ 🌤️ 天气实况 ------")
        lines.append(f"  {icon} 天气状况: {desc}")
        lines.append(f"  🌡️  温度: {temp}°C  (体感 {feels}°C)")
        lines.append(f"  💨 风速:   {wind} km/h")
        lines.append(f"  💧 湿度:   {hum}%")
        lines.append(f"  🌧️  降水量: {precip} mm")
        lines.append(f"  ☀️  紫外线: {uv}  ({uv_level})")
        lines.append("")
        lines.append("  ------ 📊 出游评估 ------")
        lines.append(f"  综合评分: {score_n} 分  {stars}")
        lines.append(f"  评分等级: {level}")
        lines.append(f"  是否适合: {suit}")
        lines.append(f"  出游建议: {advice}")
        if risks and risks != "无明显风险":
            lines.append(f"  ⚠️  风险提示: {risks}")
        lines.append("")
        lines.append("=" * 38)

        return "\n".join(lines)
```

> **与旧Shell脚本对比**：
> | 维度 | 旧 SH_TRAVEL_NOTIFY (Shell) | 新 PY_TRAVEL_NOTIFY (Python) |

> | 输出内容 | 只回显原始JSON | 格式化表格+emoji+评分星级 |
> | 可读性 | ❌ 编程小白看不懂 | ✅ 任何人一眼就懂 |
> | 天气可视化 | ❌ 无 | ✅ 天气图标+文字说明 |
> | 风险提示 | ❌ 无 | ✅ 自动标注风险 |
> | 执行方式 | stdin 传参 | gRPC kwargs 传参 |

## 七、参数传递链路（以 city 在适合出游分支为例）

```
① n_start: CONST sourceValue="Beijing"
    ↓ ② n_api: UPSTREAM_OUTPUT {nodeId:"n_start", paramCode:"city"}
       → 引擎解析: nodeOutputsMap["n_start"]["city"] = "Beijing"
       → URL替换: https://wttr.in/Beijing?format=j1
       → 输出: weather_raw = {statusCode:200, body:"{...}"}
    ↓ ③ n_filter: UPSTREAM_OUTPUT {nodeId:"n_api", paramCode:"weather_raw"}
       → 引擎解析后，COLUMN模式透传
       → 输出: weather_raw (同上)
    ↓ ④ n_python: UPSTREAM_OUTPUT {nodeId:"n_filter", paramCode:"weather_raw"} + {nodeId:"n_start", paramCode:"city"}
       → gRPC: Script().execute(weather_raw="{statusCode:200, body:...}", city="Beijing")
       → Python: 解包body → 评分 → 返回dict → JSON序列化
       → 输出: analysis_result = {"城市":"Beijing","综合评分":90,...}
    ↓ ⑥ n_branch: UPSTREAM_OUTPUT {nodeId:"n_python", paramCode:"analysis_result"}
       → BranchPlugin: ${analysis_result.综合评分} → 解析JSON → 90
       → Aviator: 90 >= 60 → true → nextNodeId=n_notify
    ↓ out_0
⑦ n_notify: 
    score ← {nodeId:"n_python", paramCode:"analysis_result"} → 完整JSON字符串
    city  ← {nodeId:"n_start",   paramCode:"city"}            → "Beijing"
    → gRPC: Script().execute(score="{...}", city="Beijing")
    → Python: 解析JSON → 生成纯文本报告(emoji+星级)
    → 输出: notify_result = 格式化出行通知
    ↓
⑨ n_end: 合并 n_notify 输出 + 其他上游 → finalOutput
```

## 八、引擎执行对照

| 节点类型 | 执行插件 | 配置来源 | 输入方式 | 输出格式 |
|---------|---------|---------|---------|---------|
| 开始/结束 | StartEndPlugin | — | triggerParams / 上游合并 | 按 outputParams 取值 |
| 接口调用 | ApiPlugin | external_api_version (任务环境) | URL `${param}` 替换 | `{statusCode, body}` |
| 数据库查询 | DbPlugin | script_version (**PROD**) + datasource_db_version (任务环境) | SQL `${param}` 替换 | `{rows, rowCount}` |
| Python执行器 | ScriptPlugin→gRPC | script_version (**PROD**) | kwargs → `Script().execute()` | `{exitCode, stdout, result}` |
| Shell执行器 | ScriptPlugin→子进程 | script_version (**PROD**) | stdin JSON | `{exitCode, stdout, result}` |
| 数据过滤 | FilterPlugin | — | 上游 nodeOutputsMap | 原样或过滤后 |
| 条件分支 | BranchPlugin | — | Aviator表达式 + 上游输出 | `{nextNodeId, matchIndex}` |
| 子任务调用 | CommonTaskPlugin | task_dsl (**PROD**, publish_status=1) | 递归ExecEngine | 子任务END输出 |

## 十、执行结果智能渲染（2026-06-18 新增）

### 问题背景

Python 脚本（如 `PY_TRAVEL_NOTIFY`）可以生成带 emoji、换行(`\n`)、Unicode 框线的自然语言报告：

```python
lines.append("========== 🌍 出行通知报告 ==========")
lines.append(f"  📍 目的地: {city or '未知'}")
lines.append(f"  🌡️  温度: {temp}°C")
return "\n".join(lines)
```

但执行结果经过 **Python → gRPC → Java → JSON → DB → JSON → 浏览器** 六层序列化链路后，`\n` 被反复转义，最终在 Web UI 显示为字面量 `\n` 字符，编程小白无法阅读。

### 解决方案（三步联动）

| 层级 | 改动 | 文件 |
|------|------|------|
| **数据库** | `execution_log` 和 `node_execution_log` 增加 `text_output` 列 | `datafactory.sql` / `migration_add_text_output.sql` |
| **后端** | 新增 `OutputTextExtractor` 工具类，执行完成时自动检测并提取格式化文本 | `executor/util/OutputTextExtractor.java` |
| **后端** | `ExecutorTaskServiceImpl` 在保存日志时调用提取器，写入 `text_output` | `ExecutorTaskServiceImpl.java` |
| **前端** | 新增 `SmartOutputViewer.vue` 智能渲染组件，自动切换 JSON/文本视图 | `components/SmartOutputViewer.vue` |
| **前端** | `TaskTestModal` 测试弹窗使用 SmartOutputViewer | `TaskTestModal.vue` |
| **前端** | `ExecuteLogPage` 执行日志页使用 SmartOutputViewer | `ExecuteLogPage.vue` |
| **前端** | `ExecutionMonitorBall` 小火箭悬浮球节点 hover 显示文本预览 | `ExecutionMonitorBall.vue` |

### SmartOutputViewer 功能

1. **自动检测**：深度遍历 JSON 输出，找出所有"看起来像人类可读格式化文本"的字符串值
2. **双视图切换**：
   - `JSON视图` — 保留原有 JSON 格式化展示（技术人员需要）
   - `文本视图` — 以深色终端风格渲染格式化文本，换行/制表/emoji 完整还原（编程小白友好）
3. **智能默认**：有文本输出时默认展示「文本视图」，无文本时只显示「JSON视图」
4. **一键复制**：视图内容可一键复制到剪贴板
5. **去重合并**：相同文本块自动去重，避免重复显示

### 效果对比

**改前**（ExecuteLogPage → 输出结果）：
```
{
  "notify_result": "\n========== 🌍 出行通知报告 ==========\n\n  📍 目的地: Beijing\n..."
}
```
→ 用户看到的是转义的 `\n` 字符，完全无法阅读

**改后**（SmartOutputViewer 文本视图）：
```
========== 🌍 出行通知报告 ==========

  📍 目的地: Beijing

  ------ 🌤️ 天气实况 ------
  ☀️ 天气状况: Sunny
  🌡️  温度: 22°C  (体感 20°C)
  💨 风速:   15 km/h
  💧 湿度:   48%

  ------ 📊 出游评估 ------
  综合评分: 90 分  ⭐⭐⭐⭐⭐
  评分等级: 极佳
  是否适合: 是
  出游建议: 非常适合出游，祝旅途愉快！
```
→ 一目了然，编程小白也能读懂

### 文本提取算法

`OutputTextExtractor` 的判断规则：
1. 叶子字符串值长度 ≥ 30 字符
2. 包含 ≥ 2 个真实换行符
3. 不以 `{` 或 `[` 开头（排除纯 JSON）
4. 包含足够的中文/符号/emoji 信号（文本评分 ≥ 10）

只有同时满足以上条件的字符串才会被提取为"格式化文本"。普通数据值（如简短字符串、数字、JSON 对象）不受影响。


新安装无需手动迁移，`datafactory.sql` 建表语句已包含 `text_output` 列。
