# 组件参数配置清单

> 每个组件按这三层来配：**① 组件属性** → **② 节点输入参数** → **③ 节点输出参数**。
> "—" 表示该项不用填或不存在。

  ① START       — 传入 a, b, op
  ② BRANCH      — 分支列表里每行: expression + 选节点（配置即路由）

# ① START 节点

**组件编码**: `COMP_START`
**作用**: 传入两个操作数和一个运算符，发给下游

## 组件属性

| 字段 | 填什么 |
|------|--------|
| — | START 无组件属性 |

## 输入参数

| 参数编码 | 参数名称 | 数据类型 | 来源类型 | 来源值 | 这一项填什么 |
|---------|---------|---------|---------|--------|-------------|
| a | 左操作数 | NUMBER | CONST | 10 | 第一个运算数，测试可改成别的数字 |
| b | 右操作数 | NUMBER | CONST | 5 | 第二个运算数 |
| op | 运算符 | STRING | CONST | + | `+` `-` `*` `/` 四选一，用来验证分支路由 |

> **CONST** = 写死一个固定值，不从别的节点拿。

## 输出参数

| 参数编码 | 参数名称 | 数据类型 | 这一项填什么 |
|---------|---------|---------|-------------|
| a | 左操作数 | NUMBER | 曝光给下游的 SCRIPT 用 `start.a` 引用 |
| b | 右操作数 | NUMBER | 同上 |
| op | 运算符 | STRING | 曝光给 BRANCH 的表达式 `${op}` 取值 |

# ② BRANCH 节点

**组件编码**: `COMP_BRANCH`
**作用**: 根据上游传来的 `op` 值，选择走哪条脚本分支

## 组件属性（在属性面板填）

BRANCH 有三个组件属性，分成两类：

### 第一类：写表达式 → `branches`（分支表达式）

| field_code | field_name | 控件 | 填法 |
|-----------|-----------|------|------|
| branches | 分支表达式 | BRANCH_EXPR_LIST | 每行一个 Aviator 表达式，**只写表达式，不选节点** |

逐行填写：

| 行号 | 填的内容 | 含义 |
|------|---------|------|
| 1 | `${op} == '+'` | 运算符是 + |
| 2 | `${op} == '-'` | 运算符是 - |
| 3 | `${op} == '*'` | 运算符是 * |
| 4 | `${op} == '/'` | 运算符是 / |

### 第二类：选下游节点 → `targetNodeIds`

| field_code | field_name | 控件 | 填法 |
|-----------|-----------|------|------|
| targetNodeIds | 选择下游节点 | NODE_MULTI_SELECT | **下拉多选**，动态加载已连线的下游节点，按选择顺序与上面表达式一一对应 |

**对应关系**：

| branches 第几行 | targetNodeIds 第几个 | 选的是 |
|----------------|---------------------|--------|
| 第 1 行 `${op} == '+'` | 第 1 个 | add（加法脚本） |
| 第 2 行 `${op} == '-'` | 第 2 个 | sub（减法脚本） |
| 第 3 行 `${op} == '*'` | 第 3 个 | mul（乘法脚本） |
| 第 4 行 `${op} == '/'` | 第 4 个 | div（除法脚本） |

都不命中时 `nextNodeId` 为空，下游全跳过。没有兜底字段。

> **设计思想**：表达式写条件，下拉选节点。两个字段独立，按索引一一对应。控件是 NODE_MULTI_SELECT 下拉多选，不是文本框。

## 画布连线

连线仍然要画（引擎靠连线做拓扑排序和阻塞传播）：

| 线 | 从 | 到 |
|----|----|----|
| e0 | start → branch1 | |
| e1 | branch1 → add | |
| e2 | branch1 → sub | |
| e3 | branch1 → mul | |
| e4 | branch1 → div | |
| e5 | branch1 → end_err | |
| ea | add → end | |
| es | sub → end | |
| em | mul → end | |
| ed | div → end | | |

## 输入参数

| 参数编码 | 参数名称 | 数据类型 | 来源类型 | 来源值 | 这一项填什么 |
|---------|---------|---------|---------|--------|-------------|
| — | — | — | — | — | BRANCH 无需输入参数，`${op}` 自动从上游取值 |

## 输出参数

| 参数编码 | 参数名称 | 数据类型 | 这一项填什么 |
|---------|---------|---------|-------------|
| nextNodeId | 命中下游节点ID | STRING | 引擎自动填入，用于跳过来命中分支 |
| matchIndex | 命中分支索引 | NUMBER | 引擎自动填入 |
| conditionResult | 条件结果 | BOOLEAN | 引擎自动填入 |

# ④⑤⑥⑦ SCRIPT 节点（四个脚本）

**组件编码**: `COMP_PYTHON_EXECUTOR`

> 四个节点配置结构完全一样，只有 scriptCode 和输出字段名不同。

## 组件属性（在属性面板填）

| field_code | field_name | 控件 | ④ add 填 | ⑤ sub 填 | ⑥ mul 填 | ⑦ div 填 |
|-----------|-----------|------|---------|---------|---------|---------|
| scriptCode | 选择脚本 | MULTI_SELECT | `add_script` | `sub_script` | `mul_script` | `div_script` |
| className | 类名 | INPUT | `Calculator` | `Calculator` | `Calculator` | `Calculator` |
| methodName | 方法名 | INPUT | `execute` | `execute` | `execute` | `execute` |
| result_var | 结果变量 | TEXTAREA | — | — | — | — |

## 输入参数（四个脚本完全一样）

| 参数编码 | 参数名称 | 数据类型 | 来源类型 | 来源值 | 这一项填什么 |
|---------|---------|---------|---------|--------|-------------|
| a | 左操作数 | NUMBER | UPSTREAM_OUTPUT | `start.a` | 从 START 的输出拿 a |
| b | 右操作数 | NUMBER | UPSTREAM_OUTPUT | `start.b` | 从 START 的输出拿 b |

> **UPSTREAM_OUTPUT** = 从上游节点拿值。`start.a` 意思是"节点 id 为 start 的输出参数 a"。

## 输出参数（四个脚本不同）

### ④ add_script

| 参数编码 | 参数名称 | 数据类型 | 对应 Python 返回字段 |
|---------|---------|---------|-------------------|
| sum | 和 | NUMBER | `sum` |
| detail | 计算明细 | STRING | `detail` |

### ⑤ sub_script

| 参数编码 | 参数名称 | 数据类型 | 对应 Python 返回字段 |
|---------|---------|---------|-------------------|
| diff | 差 | NUMBER | `diff` |
| detail | 计算明细 | STRING | `detail` |

### ⑥ mul_script

| 参数编码 | 参数名称 | 数据类型 | 对应 Python 返回字段 |
|---------|---------|---------|-------------------|
| product | 积 | NUMBER | `product` |
| detail | 计算明细 | STRING | `detail` |

### ⑦ div_script

| 参数编码 | 参数名称 | 数据类型 | 对应 Python 返回字段 |
|---------|---------|---------|-------------------|
| quotient | 商 | NUMBER | `quotient` |
| detail | 计算明细 | STRING | `detail` |

# 验证矩阵

| 改 START 的 op | BRANCH 命中行 | 激活脚本 | 该脚本输出字段 | 预期值 (a=10,b=5) | 其他三个脚本 |
|--------------|-------------|---------|-------------|------------------|------------|
| `+` | 第 1 行 | add_script | sum | **15** | SKIPPED |
| `-` | 第 2 行 | sub_script | diff | **5** | SKIPPED |
| `*` | 第 3 行 | mul_script | product | **50** | SKIPPED |
| `/` | 第 4 行 | div_script | quotient | **2** | SKIPPED |
| `%` | 不命中 | end_err | — | 兜底处理 | 全部 SKIPPED |
